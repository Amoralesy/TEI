<tree>

<tree> encodes a tree, which is made up of a root, internal nodes, leaves, and arcs from root to leaves. [19.2 Trees]
Modulenets — Graphs, Networks, and Trees
Attributesatt.global (@xml:id, @n, @xml:lang, @xml:base, @xml:space) (att.global.rendition (@rend, @style, @rendition)) (att.global.linking (@corresp, @synch, @sameAs, @copyOf, @next, @prev, @exclude, @select)) (att.global.analytic (@ana)) (att.global.facs (@facs)) (att.global.change (@change)) (att.global.responsibility (@cert, @resp))
arity arity¶gives the maximum number of children of the root and internal nodes of the tree.
Status Optional
Datatype teidata.count
ord ord¶(ordered) indicates whether or not the tree is ordered, or if it is partially ordered.
Status Required
Datatype teidata.enumerated
Legal values are:
true
indicates that all of the branching nodes of the tree are ordered.
partial
indicates that some of the branching nodes of the tree are ordered and some are unordered.
false
indicates that all of the branching nodes of the tree are unordered.
order order¶gives the order of the tree, i.e., the number of its nodes.
Status Optional
Datatype teidata.count
Note

The size of a tree is always one less than its order, hence there is no need for both a size and order attribute.
